# Security Policy

## Supported versions

Security fixes are applied to the current `main` branch until a release support policy is published.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Report it privately through the GitHub
repository security advisory flow, including reproduction steps, affected version or commit, impact,
and any proposed mitigation. Maintainers will acknowledge the report, validate impact, coordinate a
fix, and publish an advisory after users have a reasonable update window.

Security-sensitive reports must not include secrets, access tokens, or production data.
